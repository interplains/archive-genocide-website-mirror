Verify this copy

Anyone can copy this project, so don't trust a mirror until you've verified it. Every official release is signed with the project's GPG key. A tampered or fake copy cannot pass this check.

The release-signing key

Cross-check this fingerprint against the official site and the project's pinned social posts. It must match all of them:

C24E  C92B  12D6  670A  2516   065F  9B4D  5754  99AF  A53C

Verify it — the easy way

From the folder holding key.asc, SHA256SUMS, SHA256SUMS.asc, and the release files, run the script shipped with this kit:

sh verify.sh

It checks the key fingerprint, verifies the signature, and checks the file hashes. If any step fails, stop — the release is fake or tampered.

Verify it — manually

Requires gpg (GnuPG) and sha256sum — standard on Linux/macOS.

1
Import the key and confirm the fingerprint matches the one above.
gpg --import key.asc gpg --show-keys key.asc
2
Verify the signature over the checksums (expect: “Good signature”).
gpg --verify SHA256SUMS.asc SHA256SUMS
3
Check the files against the signed hashes.
sha256sum -c SHA256SUMS

How the footage is trusted

The signed SHA256SUMS pins the media torrent. Once you have that verified torrent, BitTorrent verifies every one of the 167,000+ media files piece-by-piece as they download — so an altered file simply won't match. The chain of trust:

your trusted key → signed SHA256SUMS → authentic .torrent → BitTorrent-verified media

Two rules that keep you (and sources) safe

  • Never upload footage to a mirror. Submissions go only through the project's official, verified channel. This kit has no upload form on purpose.
  • The signature is the source of truth — not how official a mirror looks. Fakes copy the design; they can't copy the key.
